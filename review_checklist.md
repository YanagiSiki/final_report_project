# final_report_project Review Checklist

本 checklist 供專案成員於每次報告檢查時依序勾選，確保內容、圖表、數據、結構與邏輯皆符合專案規範。

## 1. 專案結構與命名

- [ ] 資料夾與檔案命名皆依 AGENT.md 規範
- [ ] chapters/ 章節結構與 prompt_master_outline.md 一致
- [ ] data/、scripts/、output/ 結構完整

## 2. 章節內容檢查

- [ ] 每章節 Markdown 內容完整、主題明確
- [ ] 章節間銜接流暢，無邏輯矛盾
- [ ] 重要敘述有數據或圖表佐證

## 3. 圖片與數據比對

- [ ] 章節中所有圖片引用皆對應 output/figures/ 實際檔案
- [ ] 圖片說明與圖表內容相符
- [ ] 若圖片或數據異動，scripts/ 與 output/ 已同步更新

## 4. 自我檢視與問答

- [ ] 針對每章節進行自我問答，檢查敘述與圖表/數據是否一致
- [ ] 發現邏輯或內容問題已於原文標註，並回溯相關數據或分析腳本

## 5. 問題與修正紀錄

- [ ] 本次 review 發現問題已彙整於備忘或協作文件
- [ ] 修正建議已明確記錄，利於後續追蹤

---

> 本 checklist 可複製於每次 review 任務，並於每項完成時勾選。
> 若有異動，請務必同步更新 scripts、output 與相關章節內容。
